开发前先阅读plan.md了解开发规范，阅读task.md了解当前任务，开发完成后要记录自己开发了什么，遇到什么问题，怎么解决的踩了什么坑，以防后面ai踩同样的坑，遇到问题先问我，需要什么就问我，编译相关我自己来编译。

每次完成任务后，要反思这次开发中暴露的方法论问题（不是技术 bug，而是"为什么会反复折腾"的根因），写成反思总结追加到 task.md 末尾，并遵循以下原则：

1. 外部 API / 第三方服务：开发前先向用户确认用哪个，不自己猜。
2. 能用简单方案就别上复杂方案（如 HTTP 够了就别碰 HTTPS/TLS）。
3. 从最小可行版本开始，跑通一个再加下一个，不叠功能。
4. 改 sdkconfig 底层配置后必须提醒用户 idf.py fullclean。
