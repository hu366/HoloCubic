# 全息棱镜宠物助手 — 产品规格说明

---

## 一、用户故事

### 宠物模式

**US-1: 看到活生生的宠物**
> 作为一个用户，我打开设备后能看到一只猫在分光棱镜中活动，它会空闲发呆、会走来走去，让我感觉它是有生命的。

**US-2: 倾斜设备让宠物移动**
> 作为一个用户，当我倾斜设备时，屏幕上的猫会跟着倾斜方向滑动，就像它在平面上"滑"过去一样，让我觉得它在和我互动。

**US-3: 摇晃设备触发特殊反应**
> 作为一个用户，当我用力摇晃设备时，猫会播放一个惊吓/摔倒的动画并发出叫声，让我觉得很有趣。

**US-4: 宠物有需求需要照顾**
> 作为一个用户，我希望猫有饥饿度和心情，饿了会表现出不开心，我可以通过"喂食"和"玩耍"让它恢复，让我有照顾宠物的责任感。

**US-5: 需求在关机后依然保持**
> 作为一个用户，即使我关机一段时间再开机，猫的需求值会根据时间衰减，就像真实的宠物一样，不会因为重启而重置。

**US-6: 调节宠物音量**
> 作为一个用户，我可以通过旋转编码器调节宠物的音效音量，满足不同场景下的音量需求。

---

### 模式切换

**US-7: 一键切换模式**
> 作为一个用户，我按下旋钮就能在宠物模式和菜单模式之间切换，并且有音效和画面反馈，让我清楚知道当前处于哪个模式。

---

### 菜单模式

**US-8: 番茄钟**
> 作为一个用户，我可以在菜单中打开番茄钟，自定义工作和休息时长，看到可视化的倒计时，结束时设备会播放提示音。

**US-9: 查看天气**
> 作为一个用户，设备连接手机热点后能自动获取并显示当前天气（城市、温度、天气状况），让我出门前快速了解天气。

**US-10: 时钟与闹钟**
> 作为一个用户，设备能显示当前准确时间，我可以设置闹钟，到时间后播放铃声提醒我。

**US-11: 音乐播放器**
> 作为一个用户，我可以浏览并播放设备上的音乐文件，看到歌曲名称，通过旋转旋钮调节音量。

**US-12: 动画看板**
> 作为一个用户，我可以浏览存储卡上的动画文件并循环播放，当作桌面摆件的展示模式。

**US-13: 菜单导航**
> 作为一个用户，我可以通过倾斜设备来浏览菜单选项，按下旋钮确认选择，长按旋钮返回上一级，操作直观自然。

---

### 全息显示

**US-14: 分光棱镜正确显示**
> 作为一个用户，从分光棱镜中看到的画面是正常方向的（非镜像），文字和图像不会左右颠倒。

---

## 二、验收标准

### 宠物模式

| 编号 | 验收标准 |
|------|----------|
| AC-1 | 设备开机后默认进入宠物模式，猫显示在屏幕中央 |
| AC-2 | 猫在空闲状态下，每 5~15 秒随机切换一到两个小动作（眨眼、晃头等） |
| AC-3 | 猫每隔 30~60 秒自动进行一段完整动画（走动等） |
| AC-4 | 倾斜设备时，猫沿倾斜方向平滑移动到新位置，边缘停止，回正后逐渐归位 |
| AC-5 | 用力摇晃设备时，猫播放惊吓动画并发出对应音效，动画播完后恢复正常 |
| AC-6 | 饥饿度低于 30% 时，猫显示难过的状态和图标提示 |
| AC-7 | 用户在菜单中触发"喂食"后，猫播放吃食动画和开心音效，饥饿度恢复至 100% |
| AC-8 | 用户在菜单中触发"玩耍"后，猫播放玩耍动画和音效，开心度恢复至 100% |
| AC-9 | 设备关机 1 小时后重新开机，饥饿度和开心度各下降 6 点 |
| AC-10 | 旋转旋钮时，宠物音效音量实时变化，设备记忆音量值 |

### 模式切换

| 编号 | 验收标准 |
|------|----------|
| AC-11 | 短按旋钮，设备在宠物模式和菜单模式之间切换 |
| AC-12 | 切换时播放过渡音效，屏幕提示当前模式名称 |

### 菜单模式

| 编号 | 验收标准 |
|------|----------|
| AC-13 | 菜单显示 5 个选项：番茄钟、天气、时钟闹钟、音乐播放器、看板 |
| AC-14 | 倾斜设备上/下方向，菜单选中项跟随移动 |
| AC-15 | 短按旋钮进入选中项，长按旋钮返回上一级 |
| AC-16 | 番茄钟：默认 25 分钟工作 + 5 分钟休息，用户可修改为任意 1~99 分钟 |
| AC-17 | 番茄钟运行时显示环形进度条和剩余时间数字，每秒更新 |
| AC-18 | 番茄钟阶段结束时播放提示音，自动切换到下一阶段（工作→休息→工作） |
| AC-19 | 天气：连接 WiFi 后自动获取，显示城市名、当前温度、天气图标和描述 |
| AC-20 | 天气：获取失败时显示"获取失败"提示，不崩溃 |
| AC-21 | 时钟：开机连接 WiFi 后自动同步网络时间，显示当前时:分:秒 |
| AC-22 | 闹钟：可添加、删除闹钟，闹钟触发时播放铃声并显示提醒，旋钮按下关闭 |
| AC-23 | 音乐播放器：显示歌曲列表，选中播放，显示歌名 |
| AC-24 | 音乐播放器：播放中可暂停/继续，上一曲/下一曲切换 |
| AC-25 | 音乐播放器：旋转旋钮调节音量 |
| AC-26 | 看板：列出存储卡中的动画文件夹，选中后循环播放该动画 |
| AC-27 | 在菜单模式下旋转旋钮，调节全局音量 |

### 全息显示

| 编号 | 验收标准 |
|------|----------|
| AC-28 | 从分光棱镜中观察，所有画面内容方向正确，文字可正常阅读 |
| AC-29 | 屏幕内容（文字、动画、UI）在 240×240 范围内完整显示，不被裁切 |

## 三、边界情况

### 启动与存储

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-1 | 主存储卡未插入 | 提示"请插入存储卡"，无法进入正常模式 |
| EC-2 | 存储卡中缺少动画文件夹 | 宠物模式显示默认静态图像或纯色替代画面 |
| EC-3 | 存储卡中缺少字库文件 | 所有中文显示为方框或英文降级 |
| EC-4 | 存储卡中缺少配置文件 | WiFi 功能使用默认空值，提示用户配置 |

### 电量与功耗

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-5 | 电池电量低于 10% | 屏幕显示低电量图标或文字提示 |
| EC-6 | 设备长时间不操作 | 屏幕亮度降低，但不关机（保持宠物运行） |

### 宠物系统

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-7 | 饥饿度降至 0 | 猫持续显示难过状态，不再自动切换到开心动画 |
| EC-8 | 开心度降至 0 | 同上，猫表现萎靡 |
| EC-9 | 喂食/玩耍时需求值已满（>90%） | 仍然播放动画但提示"已经饱了/很开心了" |
| EC-10 | 连续摇晃设备 | 每次满足摇晃阈值都触发，但有冷却时间（1~2秒），避免频繁触发 |
| EC-11 | 设备平放在桌面（零倾斜） | 猫处于屏幕中央，不移动 |

### 输入冲突

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-12 | 同时旋转和倾斜设备 | 两种输入同时生效不冲突（旋转调音量，倾斜导航或移动宠物） |
| EC-13 | 旋钮卡在中间位置 | 不触发旋转事件，只有完整的一步旋转才生效 |
| EC-14 | 误触旋钮按键 | 短按和长按有明确区分（长按 > 800ms），短按才触发切换/确认 |

### 网络与天气

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-15 | WiFi 未连接时进入天气页面 | 提示"未连接网络"，不崩溃 |
| EC-16 | 手机热点关闭或超出范围 | 天气数据保留上次缓存值，标记"数据可能过期" |
| EC-17 | 天气 API 返回异常数据 | 显示"数据异常"，不影响其他功能 |
| EC-18 | 天气 API 密钥无效或配额用完 | 显示"服务不可用"，不崩溃 |

### 音乐与音效

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-19 | DFPlayer 存储卡未插入或损坏 | 所有功能正常运行，音效静默跳过不阻塞 |
| EC-20 | 音乐播放中切换到宠物模式 | 音乐停止，切换回宠物模式音效环境 |
| EC-21 | 音量调到最低（0） | 所有声音静音，UI 显示静音状态 |
| EC-22 | 音量调到最高 | 不失真，保护扬声器不过载 |
| EC-23 | 番茄钟提示音和宠物音效同时触发 | 番茄钟提示音优先，宠物音效排队等待 |

### 时间与闹钟

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-24 | 未连接 WiFi 时查看时钟 | 显示上次同步的时间（RTC 保持走时） |
| EC-25 | 多个闹钟同时触发 | 依次处理，播放一次铃声即可 |
| EC-26 | 闹钟触发时正在播放音乐 | 音乐暂停，闹钟铃声播放，关闭闹钟后恢复音乐 |

### 极端场景

| 编号 | 边界情况 | 预期行为 |
|------|----------|----------|
| EC-27 | 设备被翻倒 180° | 宠物滑到屏幕一角，显示颠倒后仍然在可见范围内 |
| EC-28 | 持续剧烈摇晃 | 每次满足冷却时间后触发，设备不死机不重启 |
| EC-29 | 存储卡读写中途断电 | 下次开机自动检测并修复文件系统，不丢宠物需求数据 |

---

## 四、非功能需求

- 宠物模式下动画帧率不低于 10fps，确保动作流畅
- 模式切换响应时间不超过 500ms（含动画过渡）
- 旋转旋钮调节音量时延迟不超过 100ms
- 从 SD 卡读取动画文件不影响 UI 响应，不出现卡顿
- 操作系统开关期间宠物需求值不丢失
